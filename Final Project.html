Problem Description¶

The Kaggle Titanic Survival Prediction is a classic Supervised Learning problem. The goal of the learning problem is to predict individual passenger survival.

Background: The sinking of the Titanic is one of the most infamous shipwrecks in history. On April 15, 1912, during her maiden voyage, the widely considered unsinkable RMS Titanic sank after colliding with an iceberg. Unfortunately, there werent enough lifeboats for everyone on board, resulting in the death of 1502 out of 2224 passengers and crew. While there was some element of luck involved in surviving, it seems some groups of people were more likely to survive than others.

Survival prediction is a simple binary classification problem with two outcomes: Survival or Perish. Such classification problems are well suited for evaluation with Supervised Learning techniques. In this problem, survival is based on individual passenger characteristics as described below.

The following project examines the findings of Exploratory Data Analysis, describes the Data Cleaning and Feature Engineering process, demonstrates the application of Supervised Learning techniques and algorithms to build various models, compares model results, and discusses the models' relative merits.

The dataset is available from kaggle.com. Data definitions and descriptions are discussed below. Utilization of the kaggle dataset has an added benefit. Typically, various holdout techniques are used to evaluate model training accuracy. Kaggle also provides an additional test set with unknown outcomes which can be submitted and scored to measure true test accuracy.

Data Dictionary

Fare: Passenger fare
Cabin: Cabin number
Embarked Port of Embarkation; C = Cherbourg, Q = Queenstown, S = Southampton

Variable Notes
Pclass: A proxy for socio-economic status (SES); 1st = Upper, 2nd = Middle, 3rd = Lower
Age: Age is fractional if less than 1. If the age is estimated, is it in the form of xx.5

Parch: The dataset defines family relations in this way...
    Parent = mother, father
    Child = daughter, son, stepdaughter, stepson

Discussion of Results

The results demonstrate some important findings. Many different techniques and algorithms can be used for supervised learning. A variety of different types of models were built and tested for training accuracy with k-fold cross-validation method. While all the models performed within the same general range based on accuracy, there were consistent differences between models. Based on accuracy scoring, these differences may seem small. However, the difference in performance even with small differences in accuracy remains important and significant. Optimization techniques were used to identify parameters for the various models. The parameter tuning demonstrated small but consistent performance improvement across all models. Model performance on previously unseen test data consistently showed lower accuracy performance across all models for both base models and parameter optimized models. Performance differences are likely due to overfitting to training data. The various models used are susceptible to overfitting. The relatively low number of observations in the training data (891) may exaggerate any overfitting. Parameter optimization and tuning may actually exacerbate overfitting further. Overall, and not surprisingly, Random Forest seemed to reliably be the best performer, in close competitiong with Support Vector Machines. Ultimately, the models performed well with a final result placing in the top 5% of the kaggle competition leaderboard.

Lessons and Takeaways

The project highlights some important aspects about developing supervised learning models. Understanding, exploring, and cleaning data is a key component to successfully building supervised learning models. Data exploration and pre-processing may often make up the majority of the work for any project, and proved to be true with this project as well. But it demonstrates the importance and necessity of thorough data preparation prior to building models. Some of the key points for Data Cleaning demonstrated by this project were assessing data for data types and missing values, imputing missing values, normalizing or categorizing variables, and the importance of feature engineering. The Exploratory Data Analysis showed the importance of understanding relationships within the data and the insights it can provide prior to initiating model building. This project showed an important risk in the process of model building and optimization with the potential for overfitting. While some steps can be taken to mitigate overfitting, it may not always be eliminated and may even be expected. Model building and optimization can lead to further overfitting and it is important for the investigator to keep this in mind throughout the process - not rely entirely on training scores, and validate outcomes on testing. Feature Engineering can play a crucial role in model performance. This project well highlights the importance of feature engineering. This particular data set has a limited number of features. Some may seem vague or of unclear importance at first. However, different attempts with different engineered features produced significant improvements or differences in results. The attempts at Feature Engineering performed in this project highlight the flexibility and variability that comes with Feature Engineering as well as its importance particularly with sparse features.

Suggested Improvements

The learning problem solutions and results performed quite well. However, there are some clear suggestions for improvement. Certain variables were excluded but could be included in future iterations, namely 'Cabin' and 'Fare'. While 'Cabin' has a very large number of missing values, this makes imputing missing values somewhat problematic. The significance of this variable is also still unclear. However, it may be valuable and a "try and see" approach may prove it to be useful. The 'Fare' variable may also be useful. Of note, it does have a rather skewed distribution and requires either transformation (such as log scale), or standardization (such as [0, 1] scale), or discretization/categorization. Conceptually, it may be a redundant proxy variable for socio-economic status which is presumably represented by 'Pclass'. Further Feature Engineering could provide an improvement to model performance and also has a very broad range of permutation and opportunities. The 'SibSp' and 'Parch' variables seem to be the most readily applicable for Feature Engineering and can be performed in many different ways either separately or combined, or further combined with other features to develop and even more complex profile for each passenger. Multiple variables could be combined to build such a profile. The training data set is somewhat imbalanced with outcomes. Over/under sampling techniques could be tested for performance improvement. "De-tuning" models to reduce overfitting could be tested for differences in improvement. Also, one might consider a voting classifier model with all the other classifiers combined. Furthermore, Exploratory Data Analysis could be perormed on the test set to identify variations withing the set or deviations from the training set which might not have been captured in the intial iterations of data pre-processing and training.

Conclusion
The project was a two-fold success. It produced a well performing Supervised Learning model for a binary classification problem. And, the project was beneficial in developing a standardized approach and framework for predictive modeling through Exploratory Data Analysis, Data Cleaning, Model Building, Feature Engineering, and Testing.
